# CSCI-6040---Final-Project---SQUiD-Reproduction
 Synthesizing Relational Databases from Unstructured Text ​

Reproduction and noise sensitivity analysis of [SQUiD: Synthesizing Relational Databases from Unstructured Text](https://arxiv.org/abs/2505.19025) (Sadia et al., EMNLP 2025).

---

## Overview

SQUiD is a neurosymbolic framework that converts raw unstructured text into a fully formed SQLite relational database using a four-stage pipeline: Schema Generation → Value Identification → Table Population → Database Materialization.

This project:
1. **Reproduces** the paper's core results, verifying two hypotheses about SQUiD's performance and integrity compared to a single-shot GPT-4 baseline.
2. **Extends** the paper with a noise sensitivity analysis — testing how SQUiD and the GPT-4 baseline degrade when the input text contains contradictory facts, missing values, duplicate entities, or structural errors.

## Getting Started

### Prerequisites

- Python 3.10+
- Git
- A valid OpenAI API key (for SQUiD's LLM calls and the GPT-4 baseline)
- Google Colab access (recommended for running experiments)

### 1. Clone this repo

```bash
git clone https://github.com/ckmoore2/CSCI-6040---Final-Project---SQUiD-Reproduction.git
cd CSCI-6040---Final-Project---SQUiD-Reproduction
```

### 2. Clone the official SQUiD repo

```bash
git clone https://github.com/Mushtari-Sadia/SQUiD.git
```

Then immediately record the commit hash — this goes into Table 1 of the report:

```bash
cd SQUiD && git log --oneline -1
cd ..
```

### 3. Set up your Python environment

```bash
python -m venv squid-env
source squid-env/bin/activate      # Mac / Linux
squid-env\Scripts\activate         # Windows
```

Install dependencies:

```bash
pip install -r SQUiD/requirements.txt
pip install openai python-dotenv pandas matplotlib seaborn
```

Verify everything installed:

```bash
python -c "import openai, pandas, matplotlib; print('all ok')"
```

### 4. Configure your API key

**Never paste your API key directly into a notebook cell or commit it to the repo.**

**Option A — Local development (.env file):**

```bash
cp .env.example .env
# Open .env and paste your key after OPENAI_API_KEY=
```

Load it in your scripts:

```python
from dotenv import load_dotenv
import os
load_dotenv()
api_key = os.getenv("OPENAI_API_KEY")
```

**Option B — Google Colab (recommended for experiments):**

1. Click the key icon (Secrets) in the left sidebar
2. Add a new secret: Name = `OPENAI_API_KEY`, Value = your key
3. Access it in your notebook:

```python
from google.colab import userdata
import os
os.environ["OPENAI_API_KEY"] = userdata.get("OPENAI_API_KEY")
```

## Reproducibility Notes

- **Random seed:** The noise injection script uses `RANDOM_SEED = 42`. Do not change this — it must be consistent across all runs and is documented in the report.
- **API model:** Use the same OpenAI model string for all runs. Record the exact model string (e.g. `gpt-4-0613`) in Table 1 of the report template. Different model versions produce different outputs.
- **SQUiD commit hash:** Record the hash from `git log --oneline -1` inside the SQUiD folder. This goes into the report so the grader can reproduce your exact environment.
- **Save outputs immediately:** Colab sessions reset after inactivity. After every stage run, save the output file to Google Drive before closing the notebook.

## Important — What NOT to Commit

The `.gitignore` already covers these, but be aware:

- `.env` — your API key lives here; never commit it
- `*.db` — SQLite databases can be large; share via Drive instead
- `data/clean/` and `data/noisy/` — dataset files; share via Drive
- `squid-env/` — your local virtual environment
- `__pycache__/` and `*.pyc` — Python bytecode

If you accidentally stage any of these, run `git rm --cached <filename>` before committing.
